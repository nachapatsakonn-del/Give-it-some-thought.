# 💡 Idea Canvas — Lean Canvas (4 ช่องแรก)

> ทำช่วงท้ายวัน เปลี่ยน "ของที่รันได้วันนี้" → "ไอเดียนวัตกรรมที่ต่อยอด Day 2 ได้"
> เริ่มจาก **ปัญหา** ไม่ใช่ "เรามี sensor อะไร" · ทำ 4 ช่องนี้ให้คมก่อน ที่เหลือไว้ทีหลัง

- **ชื่อ Prototype (ชั่วคราว):** โครงงานระบบเพาะเลี้ยงสาหร่ายขนนกอัจฉริยะ

---

## 1️⃣ Problem — ปัญหาคืออะไร

เขียน 2–3 ข้อ ที่มีคน "ลำบาก / เสียเวลา / พลาด" จริง:
```
- เกษตรกรตรวจพบสาหร่ายผิดปกติช้า เพราะดูด้วยตา กว่าจะเห็นสีเปลี่ยนก็ลุกลามจนเสียทั้งบ่อ
- ค่าน้ำ(ความเค็ม/pH/อุณหภูมิ) แกว่งตลอดเวลาโดยมองไม่เห็น ควบคุมเองไม่ทัน
- ความเสียหายมักเกิดตอนกลางคืนที่ไม่มีใครเฝ้า ทำให้แก้ไม่ทัน เสียทั้งบ่อ
```
**ตอนนี้คนแก้ปัญหานี้ยังไง (ของเดิม)?** ใช้สายตาและประสบการณ์เฝ้าดูเอง วัดค่าน้ำเป็นครั้งคราวด้วยมือ แล้วปรับน้ำหลังเห็นปัญหาแล้ว ซึ่งมักสายเกินไป

## 2️⃣ Customer — ใครคือผู้ใช้จริง

เจาะให้ชัด ไม่ใช่ "ทุกคน":
```
เกษตรกรเลี้ยงสาหร่ายขนนกในระบบโรงเรือน/บ่อ ขนาดกลาง ที่เสียหายบ่อย (ปีละหลายครั้ง)
และฟาร์มขนาดใหญ่หลายบ่อที่เฝ้าเองไม่ไหว — ไม่ใช่ฟาร์มเล็กที่แทบไม่เคยเสียหาย
```

## 3️⃣ Value (USP) — คุณค่าเด่นในประโยคเดียว

```
"ระบบดูแลสาหร่ายขนนกอัจฉริยะ ช่วยเกษตรกร รู้ล่วงหน้าก่อนน้ำเสียและปรับสภาพน้ำให้เอง
 ต่างจากการเฝ้าดูด้วยตา เพราะ AI ทำนายก่อนเกิดและทำงานแทนได้ตลอด 24 ชม. แม้คนไม่อยู่"
```
ตัวอย่าง: *"SafeFall ช่วยผู้สูงอายุที่อยู่คนเดียว ขอความช่วยเหลือได้ทันเมื่อล้ม — ต่างจาก smartwatch แพงๆ เพราะใช้อุปกรณ์ราคาถึงได้และไม่ต้องต่อเน็ต"*

## 4️⃣ Solution — แก้ด้วยอะไร

- **Input ที่ใช้:** Thermo Modulino + DHT11 วัดสภาพอากาศ + Webcam เก็บเป็นข้อมูล time-series (Modulino ตัวไหน / กล้อง / ไมค์)
- **Class ที่ AI แยก:** Safe & Danger
- **Output ตอบสนอง:** Pixels + Buzzer + จอ (Pixels / Buzzer / จอ / log)

---

## 🎯 MVP — ฟังก์ชันเดียวที่ต้องสาธิตให้ได้

```
ค่าจากเซนเซอร์ → โมเดล Edge Impulse → UNO Q → ทำนายระดับความเสี่ยง → ไฟ/เสียงเตือน + แสดงบนแอป
```

**1 core function ที่จะโชว์ Day 2/3:** บอกได้ว่า "Safe & Danger" พร้อมไฟเปลี่ยนสีและเสียงเตือนเมื่อเข้าระดับเสี่ยง 

> คาถา: **Build less, prove more** — ฟังก์ชันเดียวที่ทำงานจริง + เล่าได้ ดีกว่าหลายอย่างที่ทำไม่สุด
